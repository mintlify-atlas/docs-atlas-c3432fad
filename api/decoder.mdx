## Overview

The peeling decoder operates on the bipartite graph between droplets and source blocks, iteratively resolving singletons (degree-1 droplets) to recover the original source data.

Two entry points are provided:
- `peeling_check` — lightweight graph-only simulation (no payloads)
- `peeling_decode` — full payload-recovery decoder with integrated verification

---

## peeling_check

```rust
pub fn peeling_check(
    k: usize,
    droplets: &[Vec<u32>]
) -> PeelResult
```

Simulates the Luby Transform "Peeling" decoder on the bipartite graph of droplets and source blocks.

<ParamField path="k" type="usize" required>
  Number of source blocks ($K$) in the epoch
</ParamField>

<ParamField path="droplets" type="&[Vec<u32>]" required>
  Vector of droplet index lists. Each inner vector contains the sorted indices of source blocks that were XORed together.
</ParamField>

This function verifies if the current set of droplets provides enough information to recover all $K$ source blocks. It uses a queue-based approach to track the "Ripple" (available singletons).

<ResponseField name="Return" type="PeelResult">
  Summary of the graph reduction process
</ResponseField>

**Example:**

```rust
use sef::decoder::peeling_check;

// Three singletons covering all blocks → success
let droplets = vec![vec![0], vec![1], vec![2]];
let result = peeling_check(3, &droplets);
assert!(result.success);

// A chain that peels from a single seed singleton
let droplets = vec![vec![0], vec![0, 1], vec![1, 2]];
let result = peeling_check(3, &droplets);
assert!(result.success);
assert_eq!(result.decoded, 3);
```

## peeling_decode

```rust
pub fn peeling_decode(
    k: usize,
    droplets: Vec<Droplet>,
    verifier: &dyn BlockVerifier
) -> DecodeResult
```

Executes the Luby Transform (LT) peeling algorithm with integrated payload recovery and data verification.

<ParamField path="k" type="usize" required>
  Number of source blocks ($K$) expected in this epoch
</ParamField>

<ParamField path="droplets" type="Vec<Droplet>" required>
  Vector of received droplets. Ownership is transferred; droplets are consumed during decoding.
</ParamField>

<ParamField path="verifier" type="&dyn BlockVerifier" required>
  Block verification implementation for validating recovered blocks and determining their true length
</ParamField>

Iteratively identifies singleton droplets, verifies their content via the provided `BlockVerifier`, and peels them out of all connected droplets in the encoding graph. Droplets that fail `validate()` are silently disabled before the main loop begins.

### Algorithm

1. Initialize the "Ripple" (queue) with all degree-1 droplets
2. For each singleton, recover the source block and verify its hash/structure
3. If verified, XOR this block out of all other droplets that include it
4. Add any newly created singletons to the queue
5. Repeat until the ripple is empty or all $K$ blocks are recovered

<ResponseField name="Return" type="DecodeResult">
  The final output containing reconstructed source blocks and detailed diagnostics
</ResponseField>

**Example:**

```rust
use sef::decoder::{peeling_decode, BlockVerifier, VerifyError, DecodeStopReason};
use sef::droplet::Droplet;

// Trivial verifier that accepts everything at full length
struct AcceptAll;
impl BlockVerifier for AcceptAll {
    fn verify_and_len(&self, _idx: u32, c: &[u8]) -> Result<usize, VerifyError> {
        Ok(c.len())
    }
}

let blocks: Vec<Vec<u8>> = vec![vec![0xAA; 8], vec![0xBB; 8], vec![0xCC; 8]];
let droplets = vec![
    Droplet { epoch_id: 0, droplet_id: 0, indices: vec![0],
              padded_len: 8, payload: blocks[0].clone() },
    Droplet { epoch_id: 0, droplet_id: 1, indices: vec![1],
              padded_len: 8, payload: blocks[1].clone() },
    Droplet { epoch_id: 0, droplet_id: 2, indices: vec![2],
              padded_len: 8, payload: blocks[2].clone() },
];

let result = peeling_decode(3, droplets, &AcceptAll);
assert!(result.is_success());
assert_eq!(result.stop_reason, DecodeStopReason::Completed);
```

## BlockVerifier

```rust
pub trait BlockVerifier {
    fn verify_and_len(
        &self,
        block_idx: u32,
        candidate: &[u8]
    ) -> Result<usize, VerifyError>;
}
```

Trait for validating recovered source blocks and stripping adaptive padding.

### verify_and_len

Validates the `candidate` buffer against the expected data for `block_idx`.

<ParamField path="block_idx" type="u32" required>
  The source block index being verified
</ParamField>

<ParamField path="candidate" type="&[u8]" required>
  The recovered payload buffer (may contain trailing zero-padding)
</ParamField>

The returned length represents the original data size *before* zero-padding was applied. The caller uses this value to strip trailing padding from `candidate`.

<ResponseField name="Return" type="Result<usize, VerifyError>">
  - `Ok(length)`: The integrity check passed; `length` is the byte count of the actual data (excluding trailing zero-padding)
  - `Err(VerifyError)`: The data is corrupted or does not match the expected block for this index
</ResponseField>

### Implementations

#### BitcoinBlockVerifier

```rust
pub struct BitcoinBlockVerifier {
    pub expected_hashes: Vec<bitcoin::BlockHash>,
}
```

Bitcoin-specific `BlockVerifier` implementation. Uses a pre-calculated list of block hashes (from a trusted header chain) to validate XOR-peeling results.

<ResponseField name="expected_hashes" type="Vec<bitcoin::BlockHash>">
  Expected block hashes, ordered by source block index within the epoch. `expected_hashes[i]` corresponds to source block $i$.
</ResponseField>

#### SymbolVerifier

```rust
pub struct SymbolVerifier<'a> {
    pub symbol_size: usize,
    pub symbol_hashes: &'a [[u8; 32]],
}
```

`BlockVerifier` for symbol-mode decoding. Verifies fixed-size symbols against a pre-computed SHA-256 hash manifest.

<ResponseField name="symbol_size" type="usize">
  Expected byte length of every symbol
</ResponseField>

<ResponseField name="symbol_hashes" type="&[[u8; 32]]">
  SHA-256 digests indexed by source block position
</ResponseField>

## PeelResult

```rust
#[derive(Debug, Clone)]
pub struct PeelResult {
    pub decoded: usize,
    pub total: usize,
    pub iterations: usize,
    pub success: bool,
}
```

Result of a "peeling" decoding attempt (from `peeling_check`).

<Expandable title="Fields">
  <ResponseField name="decoded" type="usize">
    Number of source blocks successfully recovered. Monotonically increases during the peeling process.
  </ResponseField>

  <ResponseField name="total" type="usize">
    Total number of source blocks ($K$) in the epoch
  </ResponseField>

  <ResponseField name="iterations" type="usize">
    Number of peeling steps performed (one per processed singleton). Equals `decoded` in the absence of duplicate singletons.
  </ResponseField>

  <ResponseField name="success" type="bool">
    Whether all `total` blocks were recovered (decoded == total)
  </ResponseField>
</Expandable>

## DecodeResult

```rust
#[derive(Debug)]
pub struct DecodeResult {
    pub blocks: Vec<Option<Vec<u8>>>,
    pub k: usize,
    pub decoded_count: usize,
    pub iterations: usize,
    pub verify_failures: usize,
    pub stop_reason: DecodeStopReason,
    pub failures: Vec<BlockFailure>,
}
```

The final output of the fountain code decoding process.

<Expandable title="Fields">
  <ResponseField name="blocks" type="Vec<Option<Vec<u8>>>">
    The reconstructed source blocks. `Some(data)` contains the verified, un-padded block bytes. `None` indicates the block remained an "unsolved" node in the graph.
  </ResponseField>

  <ResponseField name="k" type="usize">
    Total source blocks ($K$) expected in this epoch
  </ResponseField>

  <ResponseField name="decoded_count" type="usize">
    Count of successfully recovered and verified blocks
  </ResponseField>

  <ResponseField name="iterations" type="usize">
    Total peeling steps performed
  </ResponseField>

  <ResponseField name="verify_failures" type="usize">
    Number of droplets rejected by the `BlockVerifier`. A non-zero value suggests network corruption or an adversarial sender.
  </ResponseField>

  <ResponseField name="stop_reason" type="DecodeStopReason">
    The condition that caused the decoder to terminate (`Completed` or `Stalled`)
  </ResponseField>

  <ResponseField name="failures" type="Vec<BlockFailure>">
    A list of specific blocks that could not be recovered and their final residual degree in the graph
  </ResponseField>
</Expandable>

### Methods

#### is_success

```rust
pub fn is_success(&self) -> bool
```

Returns `true` if every source block in the epoch was recovered.

## DecodeStopReason

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum DecodeStopReason {
    Completed,
    Stalled,
}
```

Represents the termination state of the peeling decoder.

<Expandable title="Variants">
  <ResponseField name="Completed">
    Success: All $K$ source blocks were successfully recovered
  </ResponseField>

  <ResponseField name="Stalled">
    Failure: The "Ripple" of degree-1 droplets is empty, but some blocks are still missing. The graph is currently unsolvable.
  </ResponseField>
</Expandable>

## BlockFailure

```rust
#[derive(Debug, Clone)]
pub struct BlockFailure {
    pub index: u32,
    pub referenced_by: usize,
}
```

Detailed information about a source block that remained unsolved.

<Expandable title="Fields">
  <ResponseField name="index" type="u32">
    The source block index $[0, K)$ that was not recovered
  </ResponseField>

  <ResponseField name="referenced_by" type="usize">
    Residual degree: number of un-peeled droplets still referencing this block. A value of 0 means the block was never covered by any droplet in the input set.
  </ResponseField>
</Expandable>

## VerifyError

```rust
#[derive(Debug)]
pub enum VerifyError {
    Deserialize(String),
    HashMismatch {
        block_idx: u32,
        expected: String,
        got: String,
    },
    NonZeroPadding,
}
```

Errors produced when a `BlockVerifier` rejects a recovered source block.

<Expandable title="Variants">
  <ResponseField name="Deserialize(String)">
    The recovered bytes cannot be parsed as the expected structure
  </ResponseField>

  <ResponseField name="HashMismatch">
    Parsed successfully but the content hash does not match. Contains the block index, expected hash, and actual hash.
  </ResponseField>

  <ResponseField name="NonZeroPadding">
    Trailing bytes after the parsed structure are not all zero
  </ResponseField>
</Expandable>
