title: "epoch"
description: "Epoch-level configuration and utility functions for the encoder and decoder pipelines"
---

An *epoch* is a contiguous group of $K$ blockchain blocks that are fountain-encoded as a unit. This module provides [`EpochConfig`](#epochconfig) (the user-facing configuration struct passed to CLI commands), [`compute_epoch_seed`](#compute_epoch_seed) for deterministic seed derivation, and [`auto_scale_droplets`](#auto_scale_droplets) for heuristic droplet-count selection.

## Example

```rust
use sef::epoch::{compute_epoch_seed, auto_scale_droplets};

let seed = compute_epoch_seed(0, "000000000019d6689c085ae165831e93");
let n = auto_scale_droplets(100, 0); // auto: 100 * 3 = 300
assert_eq!(n, 300);
```

## EpochConfig

```rust
#[derive(Debug, Clone)]
pub struct EpochConfig {
    pub k: usize,
    pub n: u64,
    pub buffer: usize,
    pub c: f64,
    pub delta: f64,
    pub symbol_size: usize,
}
```

User-facing configuration for encoding and reconstruction of a single epoch.

Passed to CLI commands (`encode`, `decode`) and threaded through both the `crate::droplet::Encoder` and `crate::decoder` pipelines.

### Fields

<ParamField path="k" type="usize" required>
  Number of source blocks per epoch ($K$).

  Controls the Robust Soliton Distribution's probability mass function and the bipartite graph width.
</ParamField>

<ParamField path="n" type="u64" required>
  Number of droplets to generate.

  When set to `0`, [`auto_scale_droplets`](#auto_scale_droplets) selects a value based on `k`.
</ParamField>

<ParamField path="buffer" type="usize" required>
  Number of most-recent blocks to retain in raw (unencoded) form.

  These blocks remain immediately readable without a decode step and are excluded from the current epoch's encoding pass.
</ParamField>

<ParamField path="c" type="f64" required>
  Robust Soliton parameter $c$.

  Governs the height of the degree-1 spike; smaller values improve efficiency but increase decoder-stall risk.
</ParamField>

<ParamField path="delta" type="f64" required>
  Robust Soliton parameter $\delta$.

  Upper bound on the probability that decoding fails after receiving $K(1+\varepsilon)$ droplets.
</ParamField>

<ParamField path="symbol_size" type="usize" required>
  Fixed symbol size in bytes for the optional [`symbol::blocks_to_symbols`](../symbol#blocks_to_symbols) normalization pass.

  Set to `0` to disable symbol slicing and encode raw variable-length blocks directly.
</ParamField>

## compute_epoch_seed

```rust
pub fn compute_epoch_seed(epoch_idx: usize, first_block_hash: &str) -> [u8; 32]
```

Derives a deterministic 256-bit epoch seed via SHA-256 domain separation.

The hash input is `b"epoch_seed" || epoch_idx (LE-u64) || first_block_hash`. Including the first block's hash binds the seed to actual chain data, preventing seed replay across forks: two epochs with the same index on different forks will produce distinct seeds because their genesis blocks (and therefore their first-block hashes) differ.

### Parameters

<ParamField path="epoch_idx" type="usize" required>
  Zero-based index of the epoch.
</ParamField>

<ParamField path="first_block_hash" type="&str" required>
  Hex-encoded hash of the first block in this epoch.

  Binds the seed to actual chain data to prevent replay across forks.
</ParamField>

### Returns

<ResponseField name="seed" type="[u8; 32]">
  Deterministic 256-bit seed for the fountain encoder.
</ResponseField>

### Example

```rust
use sef::epoch::compute_epoch_seed;

let seed = compute_epoch_seed(0, "000000000019d6689c085ae165831e934ff763ae46a2a6c172b3f1b60a8ce26f");
println!("Epoch 0 seed: {:?}", seed);
```

## auto_scale_droplets

```rust
pub fn auto_scale_droplets(unit_k: usize, n: u64) -> u64
```

Selects the droplet count $N$ when the user requests auto-scaling (`n == 0`).

Applies tiered multipliers to the source-unit count $K$:

| $K$         | Multiplier | Rationale |
|-------------|------------|------------|
| $< 50$      | $5\times$  | Small epochs suffer proportionally higher RSD overhead; extra droplets ensure the ripple never stalls. |
| $50 – 199$  | $3\times$  | Moderate overhead; standard safety margin. |
| $\ge 200$   | $2\times$  | Large $K$ concentrates the RSD well; minimal surplus needed. |

If `n` is already nonzero it is returned unchanged.

### Parameters

<ParamField path="unit_k" type="usize" required>
  Number of source units (blocks or symbols) per epoch.
</ParamField>

<ParamField path="n" type="u64" required>
  User-requested droplet count. If `0`, auto-scaling is applied.
</ParamField>

### Returns

<ResponseField name="droplets" type="u64">
  Final droplet count. Either the user-provided `n` (if nonzero) or an auto-scaled multiple of `unit_k`.
</ResponseField>

### Example

```rust
use sef::epoch::auto_scale_droplets;

// Small epoch: 5x multiplier
assert_eq!(auto_scale_droplets(30, 0), 150);

// Medium epoch: 3x multiplier
assert_eq!(auto_scale_droplets(100, 0), 300);

// Large epoch: 2x multiplier
assert_eq!(auto_scale_droplets(250, 0), 500);

// User-specified count is preserved
assert_eq!(auto_scale_droplets(100, 1000), 1000);
```

## Design Rationale

### Why domain separation in seed derivation?

The `"epoch_seed"` prefix ensures that epoch seeds never collide with seeds used for other purposes (e.g., key derivation). Including `epoch_idx` ensures different epochs have different seeds even on the same chain. Including `first_block_hash` binds the seed to the actual chain data, preventing replay attacks across forks.

### Why tiered multipliers for auto-scaling?

The Robust Soliton Distribution's overhead scales inversely with $K$: smaller epochs require proportionally more droplets to achieve the same decoding probability. The tiered multipliers balance efficiency (minimizing storage) against reliability (ensuring successful decoding).

For $K < 50$, the RSD spike is poorly concentrated, so a $5\times$ multiplier provides high safety margin. For $K \ge 200$, the distribution is well-behaved and a $2\times$ multiplier suffices.
