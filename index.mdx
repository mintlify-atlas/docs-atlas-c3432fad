## What is SeF?

SeF replaces archival full nodes with lightweight **droplet nodes** that store only a fraction of the blockchain. Using fountain codes, the chain is encoded into coded blocks called **droplets**. Any new node can reconstruct the full blockchain by collecting droplets from multiple peers and running an error-resilient decoder.

<CardGroup cols={2}>
  <Card title="Storage Reduction" icon="hard-drive">
    Store only **s ≪ k** droplets per epoch instead of k full blocks, achieving γ = k/s compression
  </Card>
  <Card title="Error Resilience" icon="shield-check">
    Authenticated verification via Merkle roots detects and discards adversarial droplets without propagating errors
  </Card>
  <Card title="Efficient Recovery" icon="arrows-rotate">
    Reconstruct the chain with probability ≥ 1 − δ by contacting k + O(√k · ln²(k/δ)) honest nodes
  </Card>
  <Card title="Bitcoin Integration" icon="bitcoin">
    Read blocks from Bitcoin Core datadir via blk*.dat files or bitcoinkernel
  </Card>
</CardGroup>

## Quick Start

Get up and running with SeF in minutes.

<Steps>
  <Step title="Install SeF">
    Add SeF to your Rust project or build the CLI tool
    
    ```toml Cargo.toml
    [dependencies]
    sef = "0.1"
    ```
  </Step>
  
  <Step title="Encode blocks into droplets">
    Use the encoder to generate fountain-coded droplets from your blocks
    
    ```rust
    use sef::distribution::RobustSoliton;
    use sef::droplet::{Encoder, EpochParams};
    
    let k = 50;
    let params = EpochParams::new(0, k as u32, [0u8; 32]);
    let dist = RobustSoliton::new(k, 0.1, 0.05);
    let encoder = Encoder::new(&params, &dist, &blocks);
    let droplets = encoder.generate_n(k as u64 * 3);
    ```
  </Step>
  
  <Step title="Decode and verify">
    Reconstruct the original blocks using the peeling decoder
    
    ```rust
    use sef::decoder::peeling_decode;
    
    let result = peeling_decode(k, &droplets, &verifier);
    assert!(result.success);
    ```
  </Step>
</Steps>

## Architecture

SeF implements the complete fountain code pipeline:

1. **Block Source** — Read Bitcoin blocks from blk*.dat files or bitcoinkernel
2. **Epoch Partitioning** — Group blocks into epochs of k blocks
3. **Symbol Normalization** — Optionally slice variable-length blocks into fixed-size symbols
4. **Encoding** — Sample degrees from Robust Soliton Distribution and XOR blocks into droplets
5. **Network/Storage** — Distribute droplets across peer nodes
6. **Decoding** — Collect droplets and run peeling decoder with Merkle root verification
7. **Reconstruction** — Recover original blocks from decoded symbols
